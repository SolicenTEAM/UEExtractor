# Solicen.UEExtractor

[**English**](/README.md) | [**Русский**](./docs/ru/README.ru.md)

Сделано с ❤️ для **всех** переводчиков и разработчиков переводов.

Это инструмент на **.NET 9.0** для извлечения текста из любой игры на [Unreal Engine](https://www.unrealengine.com/) (4.0 - 5.8).<br>Используется [CUE4Parse](https://github.com/FabianFG/CUE4Parse) для работы с Unreal Engine архивами **`.pak`** и **`.utoc`**.

С его помощью вы получите файл `locresCSV` для локализации игры на основе её ресурсов.

> [!IMPORTANT]
> Теперь поддерживается **полноценное извлечение** `DataTable`, `StringTable` и **прямое чтение `.locres` файлов** для игр, в которых локализация хранится в заранее скомпилированных бинарных файлах.

## Структура LocresCSV:
Может быть импортирована или преобразована в файл `.locres`.
- key = хэш (строка из 32 символов)
- source = расшифрованная строка
- Translation = (Null из распакованных ресурсов)
```
key,source,Translation
4A6FDB1549E45F6C5D8D739129686E2F,Default,
```
*Импорт колонки **Translation** из CSV в `.locres` возможен через [UE4localizationsTool](https://github.com/amrshaheen61/UE4LocalizationsTool).*

## Подготовка:
### Для игр на Unreal Engine 4 до [ZenLoader](https://dev.epicgames.com/documentation/en-us/unreal-engine/zen-loader-in-unreal-engine).
Узнайте, необходим ли вашей игре **AES ключ** или нет — он может не потребоваться, и эти шаги можно пропустить.

> [!TIP]
> **Версия движка и тип игры определяются автоматически** по исполняемому файлу игры.
> - Для многих известных игр (см. [Поддерживаемые игры](#поддерживаемые-игры)) флаг `-v` вообще не нужен.
> - Вы всё равно можете переопределить версию через `-v=UE5_1`, если автоопределение даёт неверный результат.
> - Все значения версий UE берутся из библиотеки CUE4Parse; нужную версию можно узнать через FModel.

1. Вам нужен **AES ключ** для просмотра архивов и получения данных из них.
   - Вы можете найти его в интернете, если у вас его нет, или извлечь самостоятельно из ресурсов игры.
2. Положите файл `aes.txt` в главную папку игры — он должен содержать одну строку с ключом в hex.
   - Или укажите его через аргумент `--aes=<key>`. Ключ всегда должен начинаться с `0x`.

### Для игр на Unreal Engine 4-5 с [ZenLoader](https://dev.epicgames.com/documentation/en-us/unreal-engine/zen-loader-in-unreal-engine).
Для начала выполните шаги, описанные выше, и только после продолжайте.

> [!IMPORTANT]
> Вам нужно получить файл `.usmap` для доступа к игровым архивам и ресурсам.
> - Как и раньше, вы можете найти этот файл в интернете, если у вас его ещё нет.
> - Посмотрите на [nexusmods](https://www.nexusmods.com/) и форумах, посвящённых модам.

1. Положите файл `.usmap` в главную папку игры (или любую подпапку) — он будет найден автоматически.
2. Подготовка к работе завершена.

## Поддерживаемые игры

Следующие игры **автоматически определяются по имени папки или исполняемого файла** — флаг `-v` не нужен.
Специфичные форматы паков, смещения индексов и пользовательское шифрование применяются автоматически через CUE4Parse.

| Игра | Примечания |
|------|-----------|
| Neverness To Everness | Особое смещение индекса пака; файлы `.locres` зашифрованы (обрабатывается автоматически) |
| Ash Echoes | Кастомный провайдер файлов |
| Wuthering Waves / KuroGames | Формат пака с частичным шифрованием |
| inZOI | Особое смещение FPakInfo |
| Marvel Rivals | Кастомное шифрование (настраивается CUE4Parse автоматически, AES ключ не нужен) |
| Dead by Daylight | Кастомное шифрование (настраивается CUE4Parse автоматически, AES ключ не нужен) |
| FragPunk | Особая обработка глобального IoStore |
| Infinity Nikki | Кастомное шифрование (настраивается CUE4Parse автоматически, AES ключ не нужен) |
| Snowbreak: Containment Zone | — |

> [!TIP]
> Определение работает по **имени папки игры** и **имени главного `.exe`** (пробелы, дефисы и подчёркивания игнорируются).
> Если вашей игры нет в списке, укажите версию вручную через `-v=<UE_версия>`.

## Слияние CSV
> [!NOTE]
> Происходит автоматически при повторном запуске извлечения той же игры (в той же папке).
> * **Очень удобно при обновлениях игры: новые строки добавляются без потери перевода.**
>
> **Как именно это работает:**
> - Если строка содержит значения только в 2 колонках (`key` и `source`) и значение `Source` не совпадает с тем, что в `Key` — значение `Source` из прошлого файла добавляется как `Translation`. *(Например, при экспорте из UE4localizationsTool)*
> - В противном случае, если совпадает — значение из колонки `Translation` записывается в `Translation`.
> - Строки из предыдущего файла, которых нет в новом, сохраняются.

## Использование:
* [Загрузите](https://github.com/SolicenTEAM/UEExtractor/releases) и **перетащите папку** на `UEExtractor.exe`, чтобы получить `<имя>_locres.csv`.
* Или используйте `UEExtractor.exe` в командной строке с аргументами.

### UEExtractor - Unreal Engine (текстовый) извлекатель
- **Перетащите папку** с ресурсами игры на `UEExtractor.exe`, чтобы получить `<имя>_locres.csv`.
- **Перетащите `locresCSV`** на `UEExtractor.exe`, чтобы получить файл `.locres`.
- Или воспользуйтесь расширенными параметрами через терминал.

#### Извлечь *LocresCSV* из игры:
```cmd
UEExtractor.exe <путь_до_каталога> [имя_csv] [аргументы...]
```

#### Собрать *.locres* из csv файла:
```cmd
UEExtractor.exe <csv_файл> <путь_до_locres>
```

#### Перевод с помощью локальной LLM (Ollama, LM Studio, vLLM и др.):
```cmd
UEExtractor.exe <путь_до_каталога> --api:url=http://localhost:11434/v1/ --api:model=llama3 --lang:from=en --lang:to=ru
```
Работает любой сервер с OpenAI-совместимым эндпоинтом `/v1/chat/completions`. Ключ `--api` для локальных серверов не нужен.

#### Сканировать только известный внутренний путь (намного быстрее):
```cmd
UEExtractor.exe <путь_до_каталога> --path=HT/Content/Localization
```
Если вы уже знаете, где находится локализация (например, из FModel), используйте `--path`, чтобы не сканировать всю игру.

### Аргументы

| Аргумент | Краткий | Описание |
|----------|---------|----------|
| `--version=<ver>` | `-v` | Версия движка (например, `-v=UE5_6`, `-v=Stalker2`). Определяется автоматически при отсутствии. |
| `--aes=<key>` | `-a` | 32-символьный hex AES ключ (должен начинаться с `0x`). |
| `--path=<виртуальный_путь>` | `-path` | Ограничить обработку активами по указанному внутреннему пути (например, `--path=HT/Content/Localization`). Поиск без учёта регистра. |
| `--verbose` | `-vb` | Показывать детальный вывод по каждому файлу и диагностику вместо полосы прогресса. |
| `--skip-uexp` | `-s:xp` | Пропустить файлы `.uexp` при обработке. |
| `--skip-uasset` | `-s:et` | Пропустить файлы `.uasset` при обработке. |
| `--locres` | | Записать файл `.locres` после анализа. |
| `--all` | `-all` | Обрабатывать все папки в архиве (включая эффекты, меши, звуки и т.д.). |
| `--underscore` | `-un` | Не пропускать строки с подчёркиванием: **ex_string** |
| `--upper-upper` | `-up` | Пропускать строки ТОЛЬКО В ВЕРХНЕМ РЕГИСТРЕ: **ПРИМЕР** |
| `--no-parallel` | `-n:p` | Отключить параллельную обработку (медленнее; может вывести дополнительные данные). |
| `--table-format` | `-tf` | Заменить стандартный разделитель `,` на `\|`. |
| `--headmark` | `-m` | Добавить заголовок и футер в `.csv`. |
| `--auto-exit` | `-exit` | Автоматически завершить работу после всех процессов. |
| `--invalid` | `-i` | Включить невалидные данные в вывод. |
| `--qmarks` | `-q` | Принудительно добавлять кавычки вокруг текстовых строк. |
| `--hash` | `-h` | Включить хэш в ключ: `[key][hash],<string>`. |
| `--url` | `-url` | Включить путь до файла в ключ: `[url][key],<string>`. |
| `--picky` | `-p` | Строгий режим — отображает больше детальной информации по каждому файлу. |
| `--table:only:key=<имя>` | `-t:o:k` | Включать только записи, чей ключ/имя совпадает с указанным значением. |
| `--lang:from=<код>` | `-l:f` | Исходный язык для перевода (например, `en`). |
| `--lang:to=<код>` | `-l:t` | Целевой язык для перевода (например, `ru`). |
| `--api:key=<key>` | `-a:key` | API ключ для OpenRouter или любого сервера, требующего аутентификацию. |
| `--api:url=<url>` | `-a:url` | Кастомный OpenAI-совместимый базовый URL для локальной модели (например, `http://localhost:11434/v1/` для Ollama). |
| `--api:model=<model>` | `-a:model` | Модель для перевода (например, `tngtech/deepseek-r1t2-chimera:free` для OpenRouter или `llama3` для Ollama). |
| `--update` | | Проверить наличие новой версии на GitHub и обновиться при необходимости. |
| `--help` | | Показать справочную информацию. |
